# Как сделать RAG-бота для своего Telegram-канала

Пошаговый гайд на примере [@doronichev_bot](https://t.me/doronichev_bot) — бота, который отвечает в стиле автора канала, опираясь на реальный контент (интервью, посты, подкасты).

---

## Что получится

Telegram-бот, который:
- Отвечает в стиле автора канала, используя его реальные тексты
- Находит релевантные фрагменты через векторный поиск (RAG)
- Поддерживает контекст диалога
- Предлагает темы для разговора
- Работает на любом языке

Стек: Python + OpenAI GPT + ChromaDB + python-telegram-bot. Деплой на Railway.

---

## Шаг 1. Создай Telegram-бота

1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Отправь `/newbot`
3. Задай имя и username бота
4. Скопируй токен (формат: `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`)

---

## Шаг 2. Получи API-ключи

### OpenAI API Key (для генерации ответов и эмбеддингов)
1. Зайди на [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
2. Создай новый ключ
3. Пополни баланс (минимум $5)

### Anthropic API Key (для очистки данных, опционально)
1. Зайди на [console.anthropic.com](https://console.anthropic.com)
2. Создай ключ — он нужен только для скрипта очистки субтитров

---

## Шаг 3. Подготовь данные

Создай папку `data/` и помести туда `.txt` файлы с контентом автора:

### Откуда брать данные

| Источник | Как получить |
|----------|-------------|
| YouTube-интервью | Скачай субтитры через [downsub.com](https://downsub.com) или YouTube Studio |
| Подкасты | Транскрибируй через Whisper или любой ASR-сервис |
| Telegram-канал | Экспорт: Telegram Desktop → три точки на канале → Export Chat History → Format: HTML. Затем распарси HTML в текст |
| Посты/статьи | Скопируй текст в `.txt` файлы |

### Важно
- Один файл = один источник (интервью, набор постов и т.д.)
- Кодировка UTF-8
- Чем больше данных — тем лучше ответы. 5-10 источников — хороший минимум

---

## Шаг 4. Очисти данные

YouTube-субтитры обычно грязные: нет пунктуации, мусорные теги `[музыка]`, таймкоды. Скрипт `clean_data.py` автоматически определяет грязные файлы и прогоняет их через Claude Haiku для очистки.

```bash
# Установи зависимости
pip install anthropic python-dotenv

# Задай ключ
export ANTHROPIC_API_KEY=sk-ant-...

# Запусти очистку
python clean_data.py
```

Результат — папка `data_clean/` с чистыми текстами. Если данные уже чистые (посты, статьи), скрипт применит только базовую очистку.

---

## Шаг 5. Проиндексируй данные в векторную базу

Скрипт `rag_preparation.py` разбивает тексты на чанки, создаёт эмбеддинги через OpenAI и сохраняет в ChromaDB.

```bash
# Задай ключ OpenAI
export OPENAI_API_KEY=sk-...

# Первый запуск (или пересоздание базы с нуля)
python rag_preparation.py --rebuild

# Добавление новых данных (без удаления старых)
python rag_preparation.py
```

Скрипт автоматически берёт файлы из `data_clean/` (если есть) или `data/`.

Результат — папка `doronichev_vectordb/` с векторной базой.

### Параметры индексации (в `rag_preparation.py`)
- `chunk_size=1000` — размер чанка в символах
- `chunk_overlap=200` — перекрытие между чанками
- `text-embedding-3-large` — модель эмбеддингов (можно заменить на `text-embedding-3-small` для экономии)

---

## Шаг 6. Напиши системный промпт

Файл `system_prompt.txt` — ключевой для качества ответов. Он определяет личность бота.

### Структура промпта

```
Определи язык вопроса пользователя и отвечай на ТОМ ЖЕ языке.

## Кто ты
[Биография автора: имя, опыт, ключевые достижения]

## Как ты отвечаешь
[Стиль речи: тон, характерные фразы, структура ответа]

## Длина и структура ответа
[Правила по длине и формату]

## Диалог
[Как вести диалог: вопросы, вовлечение]

## Обращение
[Как обращаться к пользователю]

## Границы
[Темы, на которые бот НЕ отвечает]

## Примеры твоего стиля
[2-3 примера вопрос→ответ в стиле автора]
```

### Советы
- Прочитай 10-20 постов автора и выпиши характерные обороты
- Добавь 2-3 few-shot примера (вопрос → ответ в стиле автора)
- Укажи границы — о чём бот НЕ говорит
- Не перегружай промпт — 1-2 страницы максимум

## Шаг 7. Настрой конфиг

Файл `config.py` — все параметры в одном месте:

```python
import os

# Модель и генерация
MODEL = "gpt-4.1"           # Модель OpenAI (gpt-4o для экономии)
MAX_TOKENS = 2000            # Максимальная длина ответа
TEMPERATURE = 0.7            # Креативность (0.0-1.0)

# RAG
N_RESULTS = 3                # Сколько чанков из базы подставлять в контекст
EMBEDDING_MODEL = "text-embedding-3-large"

# История диалога
MAX_HISTORY_MESSAGES = 10    # Максимум сообщений в памяти
HISTORY_CONTEXT_MESSAGES = 6 # Сколько отправлять в контекст GPT

# Фильтр тем
FORBIDDEN_TOPICS = ["политика", "религия"]

# Пути
VECTOR_DB_PATH = "./doronichev_vectordb"
COLLECTION_NAME = "doronichev_knowledge"
SYSTEM_PROMPT_FILE = "system_prompt.txt"

# Аналитика
ENABLE_ANALYTICS = True
ANALYTICS_FILE = "bot_analytics.json"

# Администратор (Telegram ID)
_admin_id = os.getenv("ADMIN_USER_ID")
ADMIN_USER_ID = int(_admin_id) if _admin_id else None

# Rate limiting
RATE_LIMIT_MESSAGES = 5      # Сообщений за окно
RATE_LIMIT_WINDOW = 60       # Окно в секундах
```

### Что настроить под себя
- `MODEL` — `gpt-4.1` (лучшее качество) или `gpt-4o` (дешевле)
- `FORBIDDEN_TOPICS` — темы, на которые бот не отвечает
- `COLLECTION_NAME` — название коллекции в ChromaDB (замени на своё)
- `VECTOR_DB_PATH` — путь к базе (замени на своё)

## Шаг 8. Настрой предложенные вопросы

Файл `suggested_questions.py` — кнопки, которые показываются при `/start`.

```python
QUESTIONS_RU = [
    {
        "teaser": "Короткий тизер для кнопки",       # до ~35 символов
        "full": "Полный развёрнутый вопрос, который отправится боту"
    },
    # ...ещё вопросы
]

QUESTIONS_EN = [
    # то же для английского
]
```

### Советы
- 6-8 вопросов в каждом языке (бот показывает 3 случайных)
- Тизеры — короткие и цепляющие (провокация, интрига)
- Полные вопросы — развёрнутые, чтобы бот дал хороший ответ

## Шаг 9. Создай `.env` файл

```bash
OPENAI_API_KEY=sk-ваш-ключ
TELEGRAM_BOT_TOKEN=ваш-токен-от-BotFather
ADMIN_USER_ID=ваш-telegram-id
ADMIN_CHAT_ID=ваш-telegram-id
```

- `ADMIN_USER_ID` — твой Telegram ID (узнать: [@userinfobot](https://t.me/userinfobot))
- `ADMIN_CHAT_ID` — куда слать уведомления о новых сообщениях (можно тот же ID)

## Шаг 10. Запусти локально

```bash
# Создай виртуальное окружение
python3 -m venv venv
source venv/bin/activate

# Установи зависимости
pip install -r requirements.txt

# Запусти
python telegram_bot.py
```

Проверь:
- `/start` — приветствие + кнопки
- Отправь вопрос — должен прийти ответ в стиле автора
- `/clear` — очистка истории
- `/stats` — статистика (только для админа)

---

## Шаг 11. Задеплой на Railway

1. Залей проект на GitHub (приватный репо)
2. Зайди на [railway.app](https://railway.app), авторизуйся через GitHub
3. New Project → Deploy from GitHub Repo → выбери репо
4. Добавь переменные окружения:
   - `OPENAI_API_KEY`
   - `TELEGRAM_BOT_TOKEN`
   - `ADMIN_USER_ID`
   - `ADMIN_CHAT_ID`
5. Railway сам подхватит `Procfile`:
   ```
   worker: python3 telegram_bot.py
   ```
6. Бот запустится автоматически

### Важно для Railway
- Векторная база `doronichev_vectordb/` должна быть в репо (она нужна при старте)
- `bot_analytics.json` добавь в `.gitignore` — он создаётся автоматически
- Railway даёт $5 бесплатного кредита в месяц

## FAQ

**Можно ли использовать Claude вместо GPT для ответов?**
Да, нужно заменить вызов `openai.chat.completions.create()` на `anthropic.messages.create()` в `telegram_bot.py`. Формат сообщений немного отличается.

**Как добавить новые данные?**
Положи `.txt` файлы в `data/`, запусти `clean_data.py`, затем `rag_preparation.py --rebuild`.

**Бот отвечает не в стиле автора.**
Проверь системный промпт — добавь больше примеров стиля. Убедись, что данные качественные (не мусорные субтитры).

**Бот галлюцинирует (выдумывает факты).**
Уменьши `TEMPERATURE` в конфиге (0.5-0.6). Добавь в системный промпт: "Не выдумывай факты и истории, которых не было".

**Как узнать свой Telegram ID?**
Напиши [@userinfobot](https://t.me/userinfobot) в Telegram.
